## Differentiation & USPs

### Primary USP: Social Media Trigger Defense

**The Problem No One Has Solved:**

Social media platforms (Instagram, TikTok, Twitter) are the #1 source of triggering content for porn addicts. Suggestive content, influencer posts, and algorithm-driven recommendations create constant exposure.

**Current Solutions:** None effectively address this.

**QuitPo's Approach:**

#### Tier 1: Friction-Based Interventions (Fully Supported)

Inspired by the "one sec" app model (57% proven reduction in usage):

1. **Trigger App Detection**: When user opens Instagram/TikTok/Twitter, intercept with:
   - Breathing exercise (10-15 seconds)
   - Mindfulness prompt
   - "Are you sure you want to open this?"

2. **Time-Aware Restrictions**:
   - Late-night (10pm-6am) additional warnings
   - Longer friction during high-risk hours
   - Bedtime mode with stricter limits

3. **Usage Tracking & Alerts**:
   - "You've been on Instagram for 30 minutes"
   - Suggest recovery activity
   - Daily/weekly reports

#### Tier 2: Content Warnings (Android Only)

Using Accessibility Services (with proper disclosure):

1. **Keyword Detection**: Scan visible text for trigger words
2. **Overlay Warnings**: Display warning when risky content detected
3. **Quick Exit**: One-tap escape to recovery content

#### Tier 3: Network-Level Protection

1. **DNS Filtering Integration**: Partner with CleanBrowsing
2. **Safe Search Enforcement**: Across all browsers
3. **Adult Content Categories**: Block at network level

## Social Media Trigger Blocking (Technical Analysis)

### iOS Capabilities

| Feature | Possible? | API/Method |
|---------|-----------|------------|
| Block apps entirely | ✅ | Screen Time API |
| Set time limits | ✅ | Screen Time API |
| Friction before opening | ✅ | Screen Time API + DeviceActivity |
| Read content in other apps | ❌ | Not allowed |
| Filter images in other apps | ❌ | Not allowed |
| Supervised mode filtering | ✅ | MDM (requires device wipe) |

**Apple's Sensitive Content Analysis Framework (iOS 17+):**
- On-device nudity detection available
- Can analyze content WITHIN your app only
- Uses same tech as Communication Safety
- Requires code-signing entitlement

### Android Capabilities

| Feature | Possible? | API/Method |
|---------|-----------|------------|
| Block apps entirely | ✅ | Device Admin / Accessibility |
| Set time limits | ✅ | Usage Stats API |
| Friction before opening | ✅ | Accessibility Service |
| Read screen text | ⚠️ | Accessibility (with disclosure) |
| Overlay warnings | ✅ | SYSTEM_ALERT_WINDOW |
| VPN-based filtering | ✅ | VPNService API |

**Google Play Policy (October 2025):**
- Accessibility Service requires explicit disclosure
- Cannot claim to be "accessibility tool" unless for disabilities
- Autonomous actions prohibited
- Risk of app removal if misused
